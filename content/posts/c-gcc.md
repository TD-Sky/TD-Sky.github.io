## 参数和选项

gcc通过后缀名区分文件。

|      后缀名      |              类型              |
|:-----------------|:-------------------------------|
| `.c/.C .cc .cxx` | C/C++源代码文件                |
| `.a`             | 由目标文件构成的档案库文件     |
| `.h`             | 程序所包含的头文件             |
| `.i/.ii`         | 经过预处理的C/C++源代码文件    |
| `.m`             | Objective-C源代码文件          |
| `.o`             | 编译后的可执行目标文件，机器码 |
| `.s`             | 汇编语言源代码文件             |
| `.S`             | 经过预编译的汇编语言源代码文件 |

---

|  阶段  | 选项 |      输入     | 输出  |
|:------:|:----:|:-------------:|:-----:|
| 预处理 | -E   |      .c       |  .i   |
| 编译   | -S   |      .i       |  .s   |
| 汇编   | -c   |      .s       |  .o   |
| 链接   |      |  .o + 库文件  | .out  |

---

|     选项      |                        描述                        |
|:--------------|:---------------------------------------------------|
| `-o filename` | 编译时指定可执行目标文件名                         |
| `-g`          | 在生成的可执行文件中包含标准调试信息               |
| `-v`          | 打印编译器内部各编译阶段的命令行信息和编译器版本号 |
| `-I dir`      | 在头文件的搜索路径列表中添加dir目录                |
| `-L dir`      | 在库文件的搜索路径列表中添加dir目录                |
| `-static`     | 链接静态库                                         |
| `-llibrary`   | 链接名为library的库文件                            |
| `Dmacro`      | 定义指定的宏，使其可通过源代码中的#ifdef进行检验   |
| `-O/-O2/-O3`  | 将优化状态打开，该选项不能与-g联合使用             |
| `-Wall`       | 在发生警告时取消编译操作                           |
| `-Werror`     | 在发生警告时取消编译操作                           |
| `-w`          | 禁止所有的报警                                     |
| `-pedantic`   | 严格要求符合ANSI标准，生成相应的警告信息           |
| `-std=c__`    | 选择C语言的标准                                    |



## 插入调试信息

`gcc -g`：在编译时插入不同等级的调试信息。生成的程序只能用于gdb调试，无法被其他调试器接受。

- g1：扩展的符号表、外部变量信息 ；仅用于**回溯跟踪**和**堆栈转储**
- g2：g1 + 行号、局部变量信息
- g3：g2 + 源代码中定义的宏

调试信息包含了程序中每个变量的类型和在可执行文件中的地址映射以及源代码行号。
